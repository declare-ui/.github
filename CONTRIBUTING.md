# Contributing to DeclareUI

Thank you for your interest in contributing! Here's how to get started.

## How to contribute

1. **Fork** the repository you want to contribute to
2. **Clone** your fork locally
3. **Create a branch** for your change: `git checkout -b feat/my-feature`
4. **Make your changes** following our conventions
5. **Test** your changes: `npm test`
6. **Push** to your fork: `git push origin feat/my-feature`
7. **Open a Pull Request** against the `main` branch

## Branch naming

- `feat/description` — new features
- `fix/description` — bug fixes
- `docs/description` — documentation
- `refactor/description` — code refactoring
- `test/description` — test additions/fixes

## Commit messages

We follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add Vue generator
fix: resolve Tailwind class extraction for nested conditionals
docs: update schema specification for computed properties
```

## Code review

All PRs require at least one approval from the founding team before merging.
We aim to review PRs within 48 hours.

## Code of Conduct

Be respectful, constructive, and inclusive. We're building something great together.
